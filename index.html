Lab Challenges

Below are some of the PortSwigger Academy labs I have successfully completed. Each challenge demonstrates my ability to identify, exploit, and mitigate web vulnerabilities using tools like Burp Suite.

Username Enumeration via Different Responses

Tool: Burp Suite

This lab involved exploiting inconsistencies in server responses during login attempts to identify valid usernames.

  • Problem Statement: Detect valid usernames based on variations in server error messages.
  • Approach: Used Burp Suite Intruder to automate login attempts and compared responses to identify valid users.
  • Key Lessons: Consistent server-side error messages are critical to prevent username enumeration vulnerabilities.

Username Enumeration via Response Timing

Tool: Burp Suite

This challenge demonstrated how response delays can reveal valid usernames when authentication mechanisms behave differently.

  • Problem Statement: Identify valid usernames by analyzing response time variations.
  • Approach: Used Burp Suite Repeater to test multiple requests and measured response delays for valid accounts.
  • Key Lessons: Authentication endpoints must respond uniformly to avoid timing-based enumeration vulnerabilities.

SQL Injection in WHERE Clause Allowing Retrieval of Hidden Data

Tool: Burp Suite

This lab involved exploiting an SQL injection flaw in the WHERE clause to retrieve hidden data that wasn’t intended to be displayed.

  • Problem Statement: Exploit an SQL injection flaw to retrieve hidden data from the database.
  • Approach: Intercepted requests using Burp Suite, tested payloads in parameters, and extracted hidden records through union-based injection.
  • Key Lessons: Use parameterized queries and proper input validation to mitigate SQL injection vulnerabilities.

SQL Injection Vulnerability Allowing Login Bypass

Tool: Burp Suite

In this challenge, I exploited a classic SQL injection vulnerability in the login form to bypass authentication and access restricted areas.

  • Problem Statement: Gain unauthorized access through SQL injection in the authentication process.
  • Approach: Manipulated the SQL query logic using payloads such as ' OR '1'='1 -- to bypass authentication.
  • Key Lessons: Always sanitize user inputs and enforce prepared statements to prevent authentication bypasses.
